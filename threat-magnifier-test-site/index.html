🔐 Section 4: Hashing Integrity Check Demo

Hover over the <script> blocks below with the Threat Magnifier extension active (Advanced Mode). The extension computes the SHA-256 hash of each inline script and compares it against a database of known malicious signatures.

📚 Concept: Cryptographic Hash Integrity
  • A hash function maps any input to a fixed-length fingerprint.
  • Even a 1-character change produces a completely different hash (avalanche effect).
  • Security tools maintain a database of hashes for known malicious payloads.
  • If a script's hash matches → DANGER.
Algorithm Output Length Security Status Use Case
SHA-256 256 bits (64 hex chars) ✅ Secure Malware signatures, file integrity
MD5 128 bits (32 hex chars) ⚠️ Broken (collision attacks) Legacy checksum, comparison only
SHA-1 160 bits (40 hex chars) ❌ Deprecated (SHAttered attack) Legacy — avoid for security use
